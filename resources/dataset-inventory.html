Dataset Inventory

Curated public datasets for the 12-week curriculum. Treasury/finance bias by design — leveraging Stephen's domain as the moat.

Tier 1 — Use Throughout Course

Chinook Database

  • URL: github.com/lerocha/chinook-database
  • License: Project docs say MIT. GitHub metadata says NOASSERTION. Verify LICENSE file before redistributing in your own public repo.
  • What it is: Sample music-store DB, 11 tables, realistic relationships (Customer → Invoice → InvoiceLine → Track → Album).
  • Why it's good: Small enough to understand, rich enough for analyst-grade questions.
  • Used in: Weeks 4, 5, 6, 10.

Northwind Database

  • URL: github.com/microsoft/sql-server-samples (Northwind subfolder)
  • License: Verify LICENSE.txt before redistribution (Microsoft custom license).
  • What it is: Classic Microsoft sample DB, wholesale/distribution.
  • Why it's good: Second schema to prove you can query databases you have not seen before.
  • Used in: Week 5.

Tier 2 — Treasury/Finance Moat

FRED — Federal Reserve Economic Data

  • URL: fred.stlouisfed.org
  • License: Public data, free API key (instant signup).
  • What's there: 840,000+ economic time series from 119 public/private orgs.
  • Why it's good: Macro context for everything. Interest rates, unemployment, inflation, GDP, yield curves.
  • Used in: Weeks 7, 11, 12.

US Treasury Fiscal Data

  • URL: fiscaldata.treasury.gov
  • License: Public domain (US government).
  • What's there: Debt-to-the-penny, daily/monthly Treasury statements, public debt, revenue collections, exchange rates.
  • Why it's good: Literal Treasury data. Perfect for the Week 12 capstone. No API key needed.
  • Used in: Week 12.

SEC EDGAR Financial Statement Data Sets

  • URL: sec.gov/data-research/sec-markets-data/financial-statement-data-sets
  • License: Public domain (US government).
  • What's there: Structured XBRL financial statement data for all publicly traded companies, updated quarterly.
  • Why it's good: Ratio analysis across hundreds of companies at once. Great for the Week 3 project.
  • Used in: Weeks 3, 11 (optional), 12.

Yahoo Finance (unofficial via yfinance)

  • URL: used via pip install yfinance
  • License: Tolerated, not official. Not guaranteed.
  • What's there: Stock prices, fundamentals, dividends.
  • Used in: Weeks 11–12 optional supplement.
  • Caveat: Yahoo can break the library anytime. Do not rely on it.

Tier 3 — Stretch / Inspiration

Kaggle Competitions Archive

kaggle.com/competitions — Licenses vary per competition. Only relevant if Stephen wants to stretch into ML in Weeks 11-12.

Stefan Jansen ML-for-Trading

github.com/stefan-jansen/machine-learning-for-trading — Last push 2024-08, stale. Use for inspiration only, not foundation.

What to Avoid

  • Any dataset Stephen has from a prior employer. That is their IP.
  • Personal financial data. PII risk.
  • Scraped data from live websites without reading the ToS. Traps. Stick to official APIs.

Datasets to Fabricate

For Weeks 2 (sheets) and 5 (joins practice), Stephen should fabricate small realistic datasets (500–1000 rows). Fabrication forces him to think about what “realistic” means — a useful domain-judgment muscle. Claude can help generate fake data with Python's faker library or a prompt template.
