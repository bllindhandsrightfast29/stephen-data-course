Week 3 — Financial Analysis

Phase 1 · Foundation Skill Focus: Validate Output Against Domain Knowledge ~12 hours
Course Progress
Week 3 of 12

You already know this content. Time value of money, IRR, NPV, working capital, 10-K analysis. The point this week is not learning finance. The point is using Claude to speed up work you already understand, and validating Claude's output with your own instinct. If a number comes back wrong, you have to know it looks wrong.

Learning Goals

  • Rebuild core financial analysis models (DCF, working capital, ratio analysis) in Claude-pair-work style
  • Drill Skill #3: validate AI output against domain knowledge, catch wrong answers fast
  • Pull real 10-K data from SEC EDGAR and run analysis on an actual public company
  • Ship a one-company financial analysis memo as this week's portfolio entry

Why This Week Matters

Claude will confidently return financial calculations that are wrong. Wrong discount rate. Wrong working capital definition. Wrong treatment of non-cash items. The math is correct. The input assumptions are subtly off.

You catch those errors because you have seen a real 10-K before. You know what the working capital number should roughly be for a large retailer. You know that if the DCF says the company is worth negative money, something is broken.

Every subsequent week uses this skill. SQL results look wrong. Dashboards look wrong. Python aggregations look wrong. This week drills the muscle on familiar ground.

AI Co-Work Pattern — Skill #3: Validate Against Domain Knowledge

The drill

Claude gives you a number. Before you use it, you run three checks:

  1. Order-of-magnitude check — is the number roughly what you'd expect? (millions vs billions, percent vs basis points, annual vs quarterly)
  2. Sign check — if it should be positive, is it? If working capital should be positive for a healthy company, is Claude's number positive?
  3. Sensibility check — given what you know about the company or problem, does this number make sense in context?

If any check fails, push back on Claude. Don't just re-prompt. Tell it specifically what you think is wrong and why.

Example of the drill in action

I asked Claude:
"Using Apple's 2023 10-K, calculate free cash flow."

Claude returned:
"$99.6B"

My order-of-magnitude check: Apple's revenue was ~$383B in 2023.
A 25% FCF-to-revenue ratio is plausible for a mature tech company.
Passes.

My sign check: Apple is profitable, FCF should be positive. Passes.

My sensibility check: I know Apple reported FCF of ~$99B in their
earnings calls. This matches.

Verdict: trust the number.

---

Different example. I asked Claude:
"Using Tesla's 2023 10-K, calculate days sales outstanding (DSO)."

Claude returned:
"147 days"

My order-of-magnitude check: 147 days of AR for a mostly direct-to-
consumer auto company? That would mean customers take 5 months to
pay. Tesla's payments mostly happen at delivery.
FAILS.

I push back:
"A 147-day DSO seems way too high for Tesla's mostly direct-to-
consumer model. Can you show me the numerator and denominator you
used? I suspect you pulled a total receivables line that includes
non-trade items."

Claude re-runs with clarified input. Returns 17 days. That passes
all three checks.

The work is in the skepticism. Claude will not volunteer that it's wrong. You have to force the re-check.

The Five Drills

Five financial-analysis topics this week. Each one uses Claude to do the calculation, then you run the three-check drill on the result.

Drill 1 — Time Value of Money (PV, FV, IRR, NPV)

I need to evaluate a potential investment. I'll invest $1M today,
receive $250K at the end of each year for 5 years. My discount
rate is 8%.

Calculate:
1. NPV
2. IRR
3. Payback period
4. Discounted payback period

Show me the formulas you're using AND the Excel formula syntax
I would use to rebuild this myself. Then I will validate against
a hand calculation.

Run the three-check drill on each number. Is the sign right? Is the magnitude plausible? Does NPV-positive tie to IRR > discount rate?

Drill 2 — Working Capital Analysis

I uploaded [Apple's / Walmart's / your pick] most recent 10-K.
Walk me through calculating:

1. Net working capital (current assets - current liabilities)
2. Days sales outstanding (DSO)
3. Days inventory outstanding (DIO)
4. Days payable outstanding (DPO)
5. Cash conversion cycle (CCC)

For each one, show me the line items you're pulling from the 10-K
so I can verify the inputs. Give the Excel formula too.

Drill 3 — Ratio Analysis

Using the same 10-K, calculate these ratios for the most recent
fiscal year:

Profitability: gross margin, operating margin, net margin, ROE, ROA
Liquidity: current ratio, quick ratio, cash ratio
Leverage: debt-to-equity, interest coverage, debt-to-EBITDA
Efficiency: asset turnover, inventory turnover

For each ratio, tell me what a "good" value looks like in this
company's industry, and flag any ratio that seems out of line.

Drill 4 — Discounted Cash Flow (Simple)

Build a simple 5-year DCF for [chosen company] using these inputs:

- Revenue growth: take the last 3 years' average
- Operating margin: take the last year's actual
- Tax rate: 25%
- Reinvestment rate: 20% of EBIT
- Terminal growth rate: 2.5%
- Discount rate (WACC): 8.5%

Show me each year's projected free cash flow, the terminal value,
and the present value. End with an implied share price using the
current share count from the 10-K.

Then: tell me which assumption is most sensitive. If I change it
by 1%, how much does the share price move?

Drill 5 — The Validation Exercise

This one is different. Claude intentionally gives you subtle wrong answers. You catch them.

I want to practice catching errors. Give me 5 financial analysis
outputs (numbers with brief context). 3 of them are plausibly
correct. 2 are subtly wrong (wrong sign, wrong units, wrong
magnitude, wrong definition). Don't tell me which is which.

For each one I'll write my check and verdict. Then tell me whether
I caught the errors.

Do this drill twice. The first time you might miss one. The second time you will be sharper. That's the point.

Real Data This Week — SEC EDGAR

Pull a real 10-K

Go to SEC EDGAR Company Search. Look up a company you're personally curious about. Or use one of these recommended starters:

  • Apple (AAPL) — clean 10-K, tech giant, FCF-rich
  • Walmart (WMT) — retail, working capital story is interesting
  • Costco (COST) — membership model, negative working capital cycle
  • Starbucks (SBUX) — mature growth, debt on balance sheet

Download the most recent 10-K as PDF. Upload to your Week 3 Claude Project. Claude can read 10-K PDFs and pull line items. You validate what it pulls against the actual filing (it's right there in the PDF).

Mini-Project — One-Company Financial Memo

Deliverable: a ~1,500-word financial analysis memo on one public company, in the voice of an analyst briefing a portfolio manager.

Structure

  1. Company snapshot (100 words) — what they do, market cap, sector
  2. Financial highlights (300 words + table) — revenue, EBITDA, FCF, debt, the ratios you calculated
  3. Working capital and cash conversion analysis (300 words) — what does the CCC tell you about how they run the business?
  4. Simple DCF (300 words + output table) — your 5-year DCF, assumptions called out, implied share price vs actual share price
  5. Three things you'd want to ask management (200 words) — the sensitivity-analysis results point at questions. Write them.
  6. Bottom line (200 words) — would you buy / hold / sell and why? Your actual opinion, not a corporate hedge.

How to build it with Claude

Start a new conversation in your Week 3 Project called “Memo Build.” Upload the 10-K. Ask Claude to draft each section in turn. You rewrite each section in your own voice, correcting any analytic errors you catch.

The intro and the bottom-line sections in particular should sound like you, not like a model. You ran the analysis. You have an opinion. Write it.

Commit Saturday

Push to stephen-data-portfolio/week_03_[ticker]_financial_memo/. Include:

  • The memo as markdown or PDF
  • A short README explaining what you analyzed and which numbers you used
  • Your calculation workbook (.xlsx or Google Sheets link)

How This Builds Forward

Week 4 switches tools. You drop spreadsheets and pick up SQL. Same analytic thinking, different syntax. The validation habit you built this week carries directly over: Claude will write SQL that returns wrong numbers, you have to spot it before your boss does.

Also: Phase 1 is now complete. You've proven you can learn a new tool with Claude (Excel), apply domain judgment to catch errors (financial analysis), and ship a portfolio piece every week. Phase 2 trusts you to move faster.

Before You Start — Setup Checklist

  • New Claude Project created: “Stephen Data Course — Week 3”
  • Project instructions updated (“completed Weeks 1-2, now doing financial analysis on real 10-K data”)
  • Target company picked, 10-K downloaded, uploaded to the Project
  • Calculation workbook from Week 2 still accessible (you'll reuse formula patterns)
  • CFI Excel Crash Course cert completed (or scheduled to finish this week)
