AI Co-Work Pattern — Skill #2: Command AI to Execute

The drill — how formula pair-work actually goes

You describe what you want in plain English. Claude returns the formula AND explains what each piece does. You paste it into your sheet. You validate against a small test case. Then you ask Claude what edge cases could break it.

The five drills — do one per day Tue through Sat

Drill 1 — Pivot Tables

Upload a treasury CSV (sample data below). Ask Claude:

I have uploaded a CSV of 500 accounts receivable invoices.
Columns: invoice_date, customer_id, customer_segment, amount_usd,
paid_date, days_outstanding.

Walk me through building a pivot table that shows total amount_usd
and count of invoices by customer_segment, with a second grouping
by aging bucket (0-30, 31-60, 61-90, 90+ days).

For each step:
1. Tell me what button to click in Google Sheets / Excel
2. Explain what that click does conceptually
3. Give me the formula equivalent if I wanted to build it without
   the pivot table UI

Then generate the final table as an artifact I can download.

You rebuild it in your own sheet. Validate against Claude's artifact. Ask Claude: “if my data doubled in size, what breaks?”

Drill 2 — XLOOKUP (the VLOOKUP killer)

Explain XLOOKUP to me by comparing it to VLOOKUP.
Show me the syntax with every argument labeled.

Then: I have a sheet with two tabs. Tab 1 is "invoices" with an
invoice_id column. Tab 2 is "customers" with customer_id and
customer_tier. I need a formula in tab 1 that looks up the tier
for each invoice's customer.

Write the XLOOKUP. Explain every argument. Tell me what happens
if the customer_id is missing from tab 2.

Drill 3 — Dynamic Arrays (FILTER, SORT, UNIQUE)

I want to build a summary section at the top of my sheet that
auto-updates as I add rows to my raw data below. Specifically:

1. A unique list of customer_segments (no duplicates)
2. For each segment, the total amount_usd
3. The list sorted by total descending

Write the formulas using FILTER, SORT, UNIQUE, and SUMIF.
Explain why dynamic arrays are better than manually dragging
formulas down. Show me what happens when I add a new row.

Drill 4 — LAMBDA (advanced)

Walk me through LAMBDA in Excel/Sheets conceptually.

Then help me build a named LAMBDA called DAYS_PAST_DUE that takes
an invoice_date and a paid_date, and returns the number of days
past due (treating unpaid as days since invoice). Show me how to
save it as a named function so I can use =DAYS_PAST_DUE(A2, B2)
in any cell.

Then write the non-LAMBDA version for comparison so I understand
when LAMBDA is worth it.

Drill 5 — Power Query / Power Pipeline Logic

I'm not going to install desktop Excel for Power Query. But I
need to understand the concept because it shows up on every
analyst job description.

Explain Power Query to me conceptually. Then show me the Google
Sheets equivalent (QUERY function, IMPORTRANGE, or Apps Script)
for the three most common Power Query use cases:

1. Combining multiple sheets with the same columns into one
2. Filtering and aggregating before loading
3. Transforming column types (text to date, etc.)

Use my uploaded CSV to build a working example of each.

Sample Treasury Dataset

Pick one, upload to your Claude Project

Three options, all free:

  1. SEC EDGAR Financial Statement Data Sets — sec.gov. Real 10-K / 10-Q line items from public companies. Use the most recent quarterly ZIP.
  2. FRED Macroeconomic Series — fred.stlouisfed.org. Treasury yield curves, Fed funds rate, FX. Download any series as CSV.
  3. Synthetic AR/AP data — ask Claude to generate a realistic 500-row invoice CSV with the columns above. Tell it the distributions you want. This is faster and the data is realistic enough for learning.

For the dashboard mini-project, the synthetic AR/AP option is probably fastest. Use real SEC data in Week 3 when you start doing financial analysis.

Mini-Project — Treasury Operations Dashboard

Your deliverable this week. Build it Thursday through Saturday. Claude builds it with you.

The Workbook

Five tabs in Google Sheets (or Excel):

  1. Raw Data — 500 rows of invoices (synthetic or SEC-derived). Don't touch this tab again once it's loaded.
  2. Aged Receivables Pivot — by customer segment and aging bucket. Uses pivot table from Drill 1.
  3. 12-Week Cash Flow Forecast — rolling forward from today's receivables, simple payment-timing assumption. Uses XLOOKUP + dynamic arrays.
  4. Variance Analysis — planned vs actual, by segment. Uses LAMBDA for the variance-percent calculation.
  5. Executive Summary — 200 words, written. What's the cash position, what's the trend, what are the three biggest risks?

How to build it with Claude

Start a fresh conversation inside your Week 2 Project called “Dashboard Build.” Upload your raw data CSV. Then:

I'm building a Treasury Operations Dashboard. I have uploaded
the raw AR data. Help me build it tab by tab.

Start with Tab 2 (Aged Receivables Pivot). Tell me the formulas
I need in Google Sheets to build this, walk me through each one,
and explain what it does. When a formula is complex, also give me
a simpler version I can understand first, then the version I
should actually use.

At the end, generate the finished pivot table as an artifact so I
can compare my work to it.

Repeat for each tab. Validate your work against Claude's artifact each time. Ask Claude to stress-test each tab (“what if I got a year of new data overnight—what breaks?”).

The Exec Summary

The 200-word written summary is the part you write yourself, not Claude. It's the Skill #5 teaser (presenting to non-technical people). You use Claude to draft. You rewrite. Your voice on the summary, every time.

Commit Saturday

Options for the portfolio:

  • Google Sheets link — share with “anyone with the link can view”, paste URL into a README in your portfolio repo
  • Excel file committed — push the .xlsx directly to stephen-data-portfolio/week_02_treasury_dashboard/ alongside a README explaining what it shows

Either works. Recruiters click through the README, not the raw file.
