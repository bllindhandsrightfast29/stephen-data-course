Week 8 — Business Dashboards

Phase 3 · Visualization Skill Focus: All Five Skills Integrated Cert: Looker Studio Qualification ~13 hours
Course Progress
Week 8 of 12

You build a real dashboard this week. Real data, real tool, real publish-to-web link Stephen-the-future-job-applicant can send to a hiring manager. The tool is Looker Studio, it's free, it runs in the browser, and it's what Google uses for its own analytics. The dashboard you build this week is the centerpiece of your portfolio.

Learning Goals

  • Connect a CSV data source to Looker Studio and build a working dashboard
  • Use calculated fields, blended data sources, and interactive filters
  • Apply Week 7's design principles to a real build
  • Integrate all five AI co-work skills in a single deliverable
  • Earn the free Looker Studio Qualification from Google Skillshop
  • Publish and share the Treasury Operations Dashboard as a public portfolio link

Why Looker Studio

Three reasons, in order of importance:

  1. Free forever, browser only. No install. No paid tier for public dashboards. You can share a dashboard by URL and anyone can view it without signing up.
  2. Industry-relevant. Most treasury and FP&A teams use either Tableau or Power BI professionally, but Looker Studio has taken serious market share since Google acquired Looker. Knowing one BI tool well means you can learn the others in a weekend.
  3. The Qualification is free and real. Google Skillshop gives you an actual cert you can add to LinkedIn.
  4. Also important: dashboards you publish on Looker Studio are public URLs. Recruiters click through. You can put the link in your resume. Tableau Public does the same but Looker Studio's editor is friendlier.

The Week in Three Tracks

Track 1 — Looker Studio Qualification (the cert)

Go to Google Skillshop, search for “Looker Studio.” Take the free qualification course. Covers data sources, report structure, calculated fields, filters. Budget ~3 hours. Passing the assessment earns you a Skillshop cert that lives on your Google account and can be added to LinkedIn.

Track 2 — Claude Dashboard Drills

Five drills, one per dashboard-building skill. Each ~30-45 min. You'll use Claude to answer “how do I do X in Looker Studio” questions while actually building in the tool.

Track 3 — The Mini-Project

The Treasury Operations Dashboard, built on real-ish data (exported from your Week 6 SQL queries), published live, linked from your portfolio.

AI Co-Work Pattern — All Five Skills

Why this week integrates everything

Building a dashboard exercises every skill you've drilled:

  1. Translate problem — what question does this dashboard answer? Who's the audience?
  2. Command AI — “how do I calculate rolling 30-day average in Looker Studio calculated field?”
  3. Validate output — do the numbers in the dashboard match the SQL you validated earlier? If not, something broke.
  4. Iterate — dashboards take 5-10 iterations before they're good. That's normal.
  5. Present to stakeholders — the 5-second test. Someone glancing at this for 5 seconds walks away with what?

The Five Drills

Drill 1 — Connect data and build basic charts

I'm in Looker Studio. I want to connect a CSV data source
(exported from my Week 6 SQL queries). Walk me through:

1. Creating a blank report and adding a data source
2. Uploading a CSV via Google Sheets as the intermediate
3. The difference between dimensions (categorical) and metrics
   (numeric) and why it matters
4. Adding my first 3 charts: a time-series, a bar chart, a big
   number

For each step, tell me what to click and what to watch out for.
Then answer this: what's the most common beginner mistake when
connecting CSV data to Looker Studio?

Drill 2 — Calculated fields

Calculated fields in Looker Studio are like Excel formulas but
built into the data source. Walk me through:

1. Syntax for a simple calculated field
2. Write the formula for: (total_deposits - total_withdrawals) /
   total_deposits, as a percentage, with NULL handling
3. Write the formula for: a CASE-style bucket (Small/Medium/Large)
   based on transaction_amount
4. The difference between a data-source-level calculated field
   and a chart-level one, and when to use each

Drill 3 — Filters, controls, and interactivity

A good dashboard is interactive. Walk me through:

1. Adding a date range filter at the top of the report
2. Adding a dropdown filter for customer_segment
3. Chart-level filters vs page-level filters vs report-level
   filters
4. The “apply to all pages” option for controls
5. A common mistake: too many filters make a dashboard confusing.
   What's a good upper limit?

Build one page of my Treasury Dashboard with these three filters
working together.

Drill 4 — Theme, layout, and polish

Apply Week 7's design principles in Looker Studio:

1. Pick a color theme and apply it report-wide (show me how)
2. Fix the default fonts (sans-serif at minimum, Inter or
   similar if available)
3. Layout: a good 4-chart dashboard layout, with where the eye
   goes first
4. Whitespace: why it matters and how to add it
5. The “styled text” feature for dashboard titles and section
   headers

Critique a draft dashboard for me. Tell me what looks amateur
and how to fix it.

Drill 5 — Publish, share, and version

Dashboards are useful only when people can see them. Walk me
through:

1. The “Share” button options: link sharing vs inviting specific
   emails
2. Embedding the dashboard in a web page (for Stephen's portfolio)
3. The “make a copy” pattern so future-Stephen can use this as
   a template
4. Refresh data: if my underlying Google Sheet updates, does the
   dashboard auto-refresh? How often?
5. Versioning: how to save iterations before making big changes

Also: what's the difference between “view” and “edit”
access, and what should I give to a recruiter's link?

Mini-Project — Treasury Operations Dashboard

Deliverable: a published Looker Studio dashboard with a public share link, plus a README explaining the dashboard and your design decisions.

The dashboard

Three pages. Each page has a clear audience and one main question it answers.

  1. Page 1: Executive Summary — for a CFO or treasurer.
    • Four big-number KPIs: total cash balance, net flow this month, count of overdue payments, customer count
    • One time-series: monthly transaction volume, 12-month trend
    • One bar chart: revenue by customer segment, current quarter
  2. Page 2: Operations Detail — for an analyst doing daily work.
    • Transaction counts by type (Deposit / Withdrawal / Fee / Transfer)
    • Top 10 accounts by transaction volume this quarter
    • Payment status breakdown by segment (stacked bar)
    • Filters: date range, segment, account type
  3. Page 3: Customer Deep Dive — for drilling into a specific customer.
    • Dropdown filter: select a customer
    • That customer's account summary (table)
    • Their transaction history time series
    • Their payment timeliness metrics

How to build it with Claude

Start a new conversation in your Week 8 Project. Upload:

  • Your Customer 360 SQL output (from Week 5) as a CSV
  • A transaction-level extract (from Week 6) as a second CSV
  • Your dashboard wireframe sketch (described in text or drawn on paper and photographed)

Ask Claude to walk you through building each page. After each page, share a screenshot of your dashboard back to Claude and ask for critique. This is Skill #4 (iterate) driving the whole build.

Portfolio entry

Push to stephen-data-portfolio/week_08_treasury_dashboard/:

  • README.md — one-page dashboard description with the shareable Looker Studio link up top
  • design_decisions.md — three paragraphs: why these three pages, why these chart types, what you'd change if you had another week
  • screenshots/ — PNG exports of each page for people who don't click links

The shareable Looker Studio link is the star. Set it to “Anyone with the link can view.” Test it in an incognito browser before committing the README.
