Week 10 — Python & Pandas

Phase 4 · Python & Capstone Skill Focus: Command AI to Execute (Python Edition) Cert: Kaggle Pandas (Week 10-11) ~12 hours
Course Progress
Week 10 of 12

Python is the language analysts write when SQL isn't enough. This week you learn enough Python to read pandas code in a job posting, not be scared of it, and use it to redo analyses you already wrote in SQL. The tool is Google Colab, it runs in the browser, and it doesn't ask you to install anything.

Learning Goals

  • Read and modify pandas code with confidence (you don't need to write it from scratch)
  • Load CSVs, clean data, compute summary statistics in a Colab notebook
  • Apply Skill #2 (command AI to execute) to yet another tool
  • Start the Kaggle Pandas course (finish in Week 11 for the cert)
  • Redo one of your Week 4-6 SQL analyses in pandas and commit both versions

Why Python and Why Colab

Every mid-level and senior analyst job description lists Python as a requirement or preferred skill. You don't need to be a software engineer. You need to be able to load a CSV, clean it, compute aggregates, and make a chart. Pandas is the library that does all of that.

Google Colab is Jupyter notebooks hosted in your browser. Free. No install. Free GPU if you ever need it (you won't this week). It's the fastest way to run Python without setting up your laptop for development.

You are not going to become a Python expert in three weeks. The goal is: you can read pandas code, understand it, modify it with Claude's help, and have a few notebooks in your portfolio. That's enough to pass a screening for analyst roles that list Python.

AI Co-Work Pattern — Skill #2 Applied (Again)

Transferring the pattern from SQL

Here's the cool part: the drill is exactly the same as Week 4 (SQL). Describe what you want in plain English. Claude writes pandas code. You read every line before running it. You paste into Colab. You check output. You ask about edge cases.

If the pattern felt solid in Week 4, it'll feel solid here. You're not learning a new AI skill. You're applying the same skill to a new tool. That's the whole point of having skills instead of memorizing syntax.

The Week in Three Tracks

Track 1 — Kaggle Pandas (the cert, part 1)

Go to Kaggle Learn Pandas. Work through the first 3-4 lessons this week. Finish the course in Week 11 for the cert. Each lesson is in a browser notebook so you're actually writing code, not watching videos.

Track 2 — Claude Pandas Drills

Five drills in Google Colab, same pattern as the SQL weeks. Each ~45 min.

Track 3 — Redo a SQL Analysis in Pandas

Pick one of your Week 4-6 SQL queries (something you validated and trust). Reimplement it in pandas. Side-by-side commit to your portfolio showing both versions.

The Five Drills

Drill 1 — Load a CSV and explore

I'm in Google Colab, brand new notebook. I want to load a CSV of
treasury transactions. Walk me through:

1. How to upload a file to Colab (drag into the file panel)
2. Import pandas, read_csv with the right parameters
3. The four commands every analyst runs first on new data:
   - df.head()
   - df.info()
   - df.describe()
   - df.shape
4. What each of those tells me and what I should watch out for
5. How to see only specific columns or the first 5 rows

Show me the code as a single notebook cell I can paste and run.
Add a comment on every line explaining what it does.

Drill 2 — Filter and sort (pandas equivalent of WHERE and ORDER BY)

Translate these SQL queries to pandas:

1. SELECT * FROM transactions WHERE amount > 1000
2. SELECT * FROM transactions WHERE transaction_type = 'Withdrawal'
   AND amount > 500
3. SELECT * FROM transactions WHERE description LIKE '%Wire%'
4. SELECT * FROM transactions ORDER BY amount DESC LIMIT 10

For each pandas version, explain:
- How the syntax differs from SQL
- The most common beginner mistake (& vs AND, quotes vs strings)
- When you'd want to use .query() method vs boolean indexing

Use the same treasury data from Week 4 (I'll export my DB Fiddle
data to CSV and upload).

Drill 3 — Groupby and aggregation (pandas equivalent of GROUP BY)

Translate these SQL queries to pandas:

1. SELECT transaction_type, COUNT(*), SUM(amount)
   FROM transactions GROUP BY transaction_type
2. SELECT customer_segment, AVG(amount)
   FROM transactions JOIN accounts JOIN customers
   GROUP BY customer_segment
3. SELECT account_id, SUM(CASE WHEN amount > 0 THEN amount ELSE 0 END) AS deposits,
          SUM(CASE WHEN amount < 0 THEN ABS(amount) ELSE 0 END) AS withdrawals
   FROM transactions GROUP BY account_id

For each:
- Show the pandas code
- Explain the .groupby() pattern and .agg() method
- Show an alternate way using pd.pivot_table and when you'd prefer it

Drill 4 — Handle dates and time series

Dates in pandas are slightly annoying at first. Walk me through:

1. Converting a string column to datetime: pd.to_datetime
2. Extracting year, month, day-of-week from a datetime column
3. Filtering to a date range (all of 2025)
4. Resampling to monthly totals: df.resample('M').sum()
5. Computing month-over-month change

Show me a complete code cell that loads my treasury CSV, converts
transaction_date, computes monthly total, and shows the 5 months
with highest transaction count.

Drill 5 — Save and export

I have a pandas dataframe with analysis results. I want to:

1. Save to CSV for my portfolio
2. Save to Excel (so I can share with non-technical people)
3. Export a summary stats table as markdown for a README
4. Save the whole notebook as a PDF for archival

Walk me through each one in Colab. What are the gotchas (encoding,
index saving, file paths in Colab)?

Mini-Project — The Side-by-Side Analysis

Deliverable: one of your Week 4-6 SQL analyses reimplemented in pandas, with both versions committed and a README showing them side by side.

Which analysis to pick

Pick something non-trivial. The Customer 360 from Week 5 is good. The period-over-period from Week 6 is better. Something that uses joins, groupby, and computed columns.

How to build it with Claude

I have a SQL query that does [describe the analysis]. Here's the
query: [paste the .sql file]

Reimplement this in pandas. Write a single Colab notebook that:
1. Loads the source CSVs (I'll upload them)
2. Does the same joins, filters, aggregations as the SQL
3. Returns a dataframe with the same columns and rows as the SQL
   result
4. Has comments mapping each pandas step to the corresponding SQL
   step

After I run it, we'll compare the output to the SQL result and
confirm they match row-for-row.

The comparison

Run both. The pandas output and the SQL output should be identical. If they're not, something is wrong in one of them. Track it down. This is Skill #3 (validate) applied.

Portfolio entry

Push to stephen-data-portfolio/week_10_sql_to_pandas/:

  • analysis.ipynb — the Colab notebook (Colab can download as .ipynb)
  • analysis.sql — the original SQL query (copy from an earlier week)
  • README.md — side-by-side comparison with notes on which is easier for what

This piece demonstrates transfer: you can read the same problem in two languages. That's a stronger signal than knowing one language deeply but not the other.
