Week 11 — Analytics Pandas

Phase 4 · Python & Capstone Skill Focus: Iterate + Validate Cert: Kaggle Pandas (finish) ~12 hours
Course Progress
Week 11 of 12

Deeper pandas this week: merge (the SQL JOIN equivalent), pivot_table, apply, and the visualization basics with matplotlib and seaborn. You finish the Kaggle Pandas cert and you get one week of practice before the capstone. Think of this as the final drill before game day.

Learning Goals

  • Use pandas merge, concat, join to combine multiple data sources
  • Build pivot tables and crosstabs that match what you'd do in Excel
  • Visualize pandas data with matplotlib and seaborn (static charts, code-generated)
  • Drill Skills #3 and #4: validate pandas output and iterate with Claude to refine
  • Finish Kaggle Pandas course (fifth of five certs)
  • Ship a multi-source pandas analysis with inline visualizations

AI Co-Work Pattern — Iterate + Validate

The pandas iteration pattern

Pandas code is where you'll iterate the most. The first version Claude writes rarely handles your specific data cleanly. Expect 3-5 iterations per non-trivial analysis.

The tight loop:

  1. Describe what you want. Claude writes pandas code.
  2. Run it. See an error? Paste the error back. Claude debugs.
  3. Run it. Output looks off? Run the validation checks from Week 3 (row count, sign, magnitude, sensibility).
  4. If validation fails, tell Claude specifically what's wrong. Iterate.
  5. If validation passes, ask Claude what edge case could break it. Add defensive code if worth it.

This pattern, repeated across 3-5 analyses this week, is what makes you good at pandas by the end.

The Five Drills

Drill 1 — Merge (the JOIN equivalent)

I have two CSVs:
- customers.csv (customer_id, name, segment)
- transactions.csv (transaction_id, customer_id, amount, date)

I want a merged dataframe with customer name and segment on each
transaction row. Walk me through:

1. pd.merge syntax
2. The four "how" options: inner, left, right, outer
3. When to use on= vs left_on/right_on (different column names)
4. The validate= parameter (1:1, 1:m, m:1, m:m) and why it saves you
5. A common bug: duplicated rows after merge because of a
   many-to-many relationship you didn't know existed

For each of the four join types, write the code and tell me what
the output row count should be (validation check).

Drill 2 — Pivot table

pd.pivot_table is the SQL crosstab/pivot equivalent. Build these:

1. Transactions by transaction_type (rows) and customer_segment
   (columns), values = SUM(amount)
2. Monthly totals (rows = month, columns = transaction_type,
   values = count)
3. Customer matrix (rows = customer_name, columns = quarter,
   values = SUM(amount))

For each:
- Write pd.pivot_table code
- Show the equivalent .groupby().pivot() version
- Explain when you'd prefer pivot_table vs groupby+unstack
- Show how to fill missing cells with 0 or a placeholder

Drill 3 — apply and lambda functions

apply() lets you run a function on every row or column. Walk me
through:

1. .apply() on a Series (one column) with a simple lambda
2. .apply() on a DataFrame with axis=1 (per-row)
3. When to use apply vs vectorized operations (performance)
4. Custom function definition (def my_func(row): ...)
5. The common anti-pattern: using apply when a vectorized
   operation would be 100x faster

Example task: for each transaction, classify it as "High-Risk"
if amount > $10000 AND transaction_type in ('Withdrawal', 'Transfer'),
else "Normal." Show both apply-based and vectorized versions,
explain which is better.

Drill 4 — Visualize with matplotlib and seaborn

Basic pandas viz is often enough for exploratory analysis. Walk me
through matplotlib and seaborn for:

1. df.plot() for quick exploration (bar, line, hist, scatter)
2. plt.subplots() for multi-panel figures
3. seaborn for prettier defaults and statistical visualizations
   (seaborn.barplot, seaborn.heatmap, seaborn.boxplot)
4. Saving figures to PNG for the portfolio

Create these 5 visualizations with my treasury data:
- Bar chart of total transaction amount by segment
- Line chart of monthly transaction volume over time
- Heatmap of transaction count by day-of-week and hour
- Box plot of transaction amount by type (shows distribution)
- Scatter plot of account balance vs transaction count per customer

Save each as PNG. Apply the design principles from Week 7 (titles,
axis labels, color).

Drill 5 — Multi-index and reshape

Multi-index dataframes are confusing at first and useful once
you get them. Walk me through:

1. What a multi-index is and why it exists
2. Creating one with .set_index() on multiple columns
3. Accessing rows: df.loc[(outer_key, inner_key)]
4. .stack() and .unstack() for reshaping wide/long
5. .reset_index() to flatten back to a regular dataframe

Example: I have monthly revenue by segment. Show me how to reshape
it from "long" (one row per segment-month) to "wide" (months as
columns, segments as rows), do year-over-year math, then reshape
back to long for saving.

Mini-Project — Multi-Source Analysis with Viz

Deliverable: a Colab notebook that pulls two or three data sources, merges them, runs an analysis that requires both merge AND pivot_table, and produces three polished matplotlib charts. Committed to portfolio.

The analysis question

Pick one (or design your own):

  • Customer profitability breakdown: merge transaction data with cost-to-serve estimates, compute profit margin per customer by segment, visualize distribution
  • Cash flow seasonality: using treasury transactions over 2 years, identify seasonal patterns per quarter, visualize as a heatmap
  • Payment timeliness degradation: merge customer master with payment history, identify which segments are getting slower to pay, visualize trend

How to build it with Claude

I'm building a [chosen analysis] in pandas. I have these data
sources: [list].

Architect the analysis as a sequence of steps:
1. Load each source, inspect, clean
2. Merge (specify the join type and keys, and the validation I
   should do)
3. Any pivot_table or groupby transformation
4. Three visualizations that each answer one sub-question
5. Written summary (bullets) of the top 3 findings

Build each step one at a time. After each step I'll run it in
Colab and share the output back. Only move to the next step when
I confirm the current step validates correctly.

Push to stephen-data-portfolio/week_11_pandas_analysis/:
