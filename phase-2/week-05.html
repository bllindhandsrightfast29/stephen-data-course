Week 5 — Joins, Aggregations, and the “It Looked Right But Wasn't” Trap

Phase 2 · SQLSkill Focus: Validate (silent-duplication edition)~14 hours
Course Progress
Week 5 of 12

INNER, LEFT, RIGHT, FULL OUTER joins — when each is right. Multi-table aggregations. The classic “duplicated rows from a one-to-many” bug and how to catch it every time.

Learning Goals

  • All four join types, each with a clear use case
  • Drill Skill #3 specifically against the silent-duplication trap
  • Ship: 16 questions across Chinook and Northwind + the “bugs I caught” reflection essay

Content Releases After Week 4

Full join-by-join drill pattern, duplication-trap examples, and cross-database question set unlock after Week 4 commit.

AI Co-Work Pattern (Preview)

Claude will sometimes write a query that returns $4.2M when the right answer is $420K — because a JOIN duplicated rows. Your job: always run SELECT COUNT(*) before and after the join, and sanity-check the totals.

Resources (Preview)

  • Kaggle Intro to SQL (remaining lessons, join-heavy)
  • SQLBolt intermediate lessons
  • Microsoft SQL Server Samples (Northwind)
  • YouTube: “Visual JOINs explained” (any of several good 10-min videos)

Mini-Project (Preview)

Cross-database analysis: 8 questions requiring 3+ table joins in Chinook, 8 in Northwind. Plus a 500-word essay: “The duplicated-row bugs I caught and how I caught them.”
